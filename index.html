Safety & Governance

Trust is not optional.

AI that touches production needs guardrails. Every Claw bot runs inside a verifiable policy envelope.

Scoped Permissions

Each Claw bot declares its tool access upfront. DevClaw can read repos but can't touch DNS. OpsClaw can deploy to staging but needs approval for production. Principle of least privilege, enforced.

Approval Gates

Side-effecting actions (deploys, emails, payments, merges) require human approval. Configurable per-bot, per-action, per-environment. Workflows pause and resume with durable tokens.

Audit Trails

Every tool call, every LLM invocation, every decision point â€” logged as structured events with timestamps, token counts, and cost. Replay any workflow. Diff any two runs. Export to your SIEM.

Evaluations

Define pass/fail criteria for any workflow output. Schema validation, regex checks, LLM-as-judge, custom scripts. Evals run automatically before results are returned.

Sandboxed Execution

Claw bots run in isolated containers with resource limits â€” CPU, memory, network, filesystem. Timeout policies kill hung tasks. Output caps prevent runaway generation.

Version Control

Agent configs, workflow definitions, and policy files are declarative YAML â€” versioned in git, reviewed in PRs, deployed through CI. Infrastructure-as-code for your AI workforce.

  • How much does OpenLobster cost?

  • How is my data handled?
    Self-hosted: your data never leaves your network. Managed cloud: data is encrypted at rest (AES-256) and in transit (TLS 1.3), stored in your chosen region, and never used for model training. You can export or delete all data at any time. SOC 2 Type II audit in progress.

  • What happens if an agent fails?
    LobsterAgent handles failures gracefully. Failed steps trigger automatic retries with backoff. If retries exhaust, the workflow pauses and escalates to a human with full context. No silent failures. No half-finished states. Workflows are resumable from any checkpoint.
